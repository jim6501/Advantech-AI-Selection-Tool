④ 後端模組（Backend — FastAPI，內部電腦執行）
🖥 app/main.py  FastAPI 應用程式入口
Static mount: /frontend → frontend/ 資料夾  |  Router 掛載: /api → selection.py、chat.py
📦 /api/submitProdType
POST　篩選產品主要 API
接收：type, portnum, items[], application
查詢：MongoDB hardware + software collections
回傳：ProductItemResponse[] (Pydantic)
🔍 /api/searchProdType
GET ?q=keyword
即時搜尋功能列表
掃描 software.feat_key / hardware.Application
回傳：SearchFeatureItem[]
🤖 /api/chat
POST　AI 問答
接收：message, context (selected_models), history
呼叫：LLM API（Claude）
回傳：answer, referenced_models, sources
📄 /api/exportReport
POST　PDF 報表匯出
接收：選型條件 + 產品清單
產生 PDF 並回傳 base64
🗂 app/api/selection.py
核心選型邏輯
base_hardware_mappings：PORT_FEATURE / PORT_SPEED / POWER key labels
safe_int() 欄位轉換
ProductItemResponse 欄位從 MongoDB 逐一 mapping
⑤ 資料層
🗄 MongoDB — hardware collection
每筆為一台產品
欄位：Model Name, PN, RJ45 GbE, Fiber 10G, Application,
Fiber Type, Fiber Connector, Power Input, Temp, URL…
🗄 MongoDB — software collection
每筆為一個軟體功能項目
欄位：feat_key, label, category, support (boolean)
category 對應 feature-selector.js 的 FS_GROUPS
📁 sfp_modules.json
靜態 JSON，存放 SFP 模組型錄
欄位：part, speed, cat, mode, bidi, conn, dist, wave, note
由 sfp-selector.js 在 SFP Guide tab 開啟時載入
⑥ 完整資料流（以「搜尋產品」為例）
使用者點「Find Matching Products」
  ↓ submitItems()                      // app.js
  ↓ detectApiBase() → API_BASE        // 決定打哪個網址
  ↓ fetch POST {API_BASE}/api/submitProdType
     ↓ [Cloudflare Tunnel 或 直連 localhost:8000]
     ↓ FastAPI router → selection.py
     ↓ MongoDB query (hardware + software)
     ↓ 建立 ProductItemResponse[] (Pydantic)
     ↓ JSON response → 瀏覽器
  ↓ data.products → currentDataList
  ↓ applySort() → 排序
  ↓ _renderCurrentView(sortedList)
     ↓ renderProductCards()             // List 模式
     ↓ renderProductTable()             // Table 模式
